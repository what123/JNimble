# 插件卸载数据清理设计

## 背景

插件管理页的卸载操作默认只停用插件并移除运行时贡献，不清理插件业务数据。为支持受控场景下的彻底卸载，卸载确认弹窗提供“同时清理插件数据表和字段”选项。

## 行为

- 默认卸载仍调用 `uninstall(pluginId)`，不删除任何数据库对象。
- 勾选清理时调用 `uninstall(pluginId, true)`，这是不可逆风险操作。
- 清理范围来自插件迁移脚本：
  - `CREATE TABLE` 生成的插件表会执行 `DROP TABLE`。
  - `ALTER TABLE ... ADD COLUMN` 增加的字段会执行 `DROP COLUMN`。
  - 插件配置的 Flyway 历史表会一并删除，便于后续重新迁移。
- 清理不会调用 Flyway `clean()`，避免误清整个 schema。

## 限制

当前清理器只识别简单 SQL 迁移里的普通表名和字段名，不处理复杂动态 SQL、schema 限定名或手写存储过程。无法识别的对象不会被自动删除。
